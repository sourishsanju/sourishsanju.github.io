Cost-Based Pathfinding and Graph Search

Objective

To design and implement an optimal route-finding system between airports based on cost, time, and distance using graph-based modeling and pathfinding algorithms such as A* and Dijkstra.

What Was Done

  • Built a directed graph network representing airports and available travel paths.
  • Fetched and processed travel data including distance, time, and cost for edges using APIs.
  • Applied A* algorithm for optimal path computation between airports based on selected metrics (cost, time, comfort).
  • Developed a custom GUI to allow users to select trip type, mode of transportation, and prioritize cost or time.

Challenges

  • Managing incomplete data from APIs and handling rate limits during real-time fetches.
  • Normalizing multi-metric data (cost vs. distance vs. time) for pathfinding optimization.
  • Ensuring scalability and robustness in graph construction as more nodes (airports) are added.

What Was Learned

  • Advanced graph modeling techniques for transportation networks.
  • Optimization of A* and uniform-cost search based on real-world parameters.
  • Handling live external API data with error tolerance and dynamic updates.
  • Building user-friendly interfaces for flexible query inputs in navigation systems.

What Was Achieved

Successfully developed an interactive pathfinding model capable of providing optimized travel routes balancing cost, time, and distance. The system proved to be adaptable for real-world route planning and demonstrated efficient search computation even for dense airport networks.
